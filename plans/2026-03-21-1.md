# Plan: Inject Movie Player Button into TMDB Movie Pages

## Overview

Inject a "Play Movie" button into TMDB movie pages (`themoviedb.org/movie/*`). When clicked, a shadcn Dialog modal opens containing an iframe video player. The TMDB movie ID is extracted from the URL to construct the iframe source.

## Current State

- `entrypoints/content.ts` exists but targets `*://*.google.com/*` with a placeholder `console.log`
- The project has React, Tailwind CSS, and shadcn/ui set up
- shadcn `Button` component already exists at `components/ui/button.tsx`
- WXT content scripts use `defineContentScript` with a `matches` array and `main()` function

## Architecture

Since we're using shadcn components (React + Tailwind), the content script needs to:

1. Use WXT's `createShadowRootUi` to mount a React app inside a shadow DOM — this isolates our Tailwind/shadcn styles from the host page and vice versa
2. The content script entrypoint becomes a `.tsx` file to support JSX
3. A shared `styles/content.css` file imports Tailwind and shadcn theme variables (reusing from `entrypoints/popup/style.css`)

The injected React component tree:
- `PlayerButton` — renders a shadcn `Button` inside the TMDB `<ul>`
- `PlayerModal` — renders a shadcn `Dialog` with the iframe, triggered by the button

Since the shadow root UI is isolated, we need a two-part approach:
- **Part A**: Inject a minimal native `<li><button>` into the TMDB `<ul>` (this lives in the host DOM, not shadow DOM, so it blends with the existing TMDB layout)
- **Part B**: Mount a React shadow root UI for the modal/dialog overlay (this is isolated and uses full shadcn styling)

This hybrid approach is necessary because shadow DOM elements cannot be children of a host DOM `<ul>` — the button must be a native DOM node in the host page, while the modal lives in our own shadow root.

## Implementation Steps

### Step 1: Install shadcn Dialog component

```bash
pnpm dlx shadcn@latest add dialog
```

This adds `components/ui/dialog.tsx` with `DialogOverlay`, `DialogContent`, `DialogClose`, etc.

### Step 2: Rename content script to `.tsx` and update match pattern

Rename `entrypoints/content.ts` → `entrypoints/content.tsx` to support JSX.

```tsx
// entrypoints/content.tsx
import "./content.css";

export default defineContentScript({
  matches: ["*://*.themoviedb.org/movie/*"],
  main(ctx) {
    const tmdbId = getTmdbId();
    if (!tmdbId) return;
    injectPlayButton(tmdbId);
    mountPlayerModal(ctx, tmdbId);
  },
});
```

### Step 3: Extract the TMDB movie ID from the URL

The URL format is `/movie/{id}-{slug}`. We need the numeric ID before the first `-`.

```ts
function getTmdbId(): string | null {
  const lastSegment = window.location.pathname.split("/").pop();
  if (!lastSegment) return null;
  const id = lastSegment.split("-")[0];
  return /^\d+$/.test(id) ? id : null;
}
```

### Step 4: Inject a native play button into the TMDB header

This button lives in the host DOM to blend with TMDB's existing `<ul>` layout. It dispatches a custom event to communicate with our shadow root React app.

```tsx
function injectPlayButton(tmdbId: string) {
  const targetUl = document.querySelector(
    "#original_header > div.header_poster_wrapper.true > section > div.flex.flex-col > ul"
  );
  if (!targetUl) return;

  const li = document.createElement("li");
  li.className = "kawakumo-play-item";

  const button = document.createElement("button");
  button.textContent = "▶ Play Movie";
  button.className = "kawakumo-play-button";
  button.addEventListener("click", () => {
    document.dispatchEvent(new CustomEvent("kawakumo:open-player"));
  });

  li.appendChild(button);
  targetUl.appendChild(li);
}
```

### Step 5: Mount the shadcn Dialog in a shadow root

Use WXT's `createShadowRootUi` to mount a React component containing the shadcn `Dialog`. The component listens for the `kawakumo:open-player` custom event to open.

```tsx
import ReactDOM from "react-dom/client";
import { createShadowRootUi } from "wxt/utils/content-script-ui";
import { PlayerModal } from "@/components/PlayerModal";

async function mountPlayerModal(ctx: ContentScriptContext, tmdbId: string) {
  const ui = await createShadowRootUi(ctx, {
    name: "kawakumo-player",
    position: "inline",
    onMount(container) {
      const root = ReactDOM.createRoot(container);
      root.render(<PlayerModal tmdbId={tmdbId} />);
      return root;
    },
    onRemove(root) {
      root?.unmount();
    },
  });
  ui.mount();
}
```

### Step 6: Create the PlayerModal React component

```tsx
// components/PlayerModal.tsx
import { useEffect, useState } from "react";
import {
  Dialog,
  DialogContent,
  DialogOverlay,
  DialogPortal,
  DialogTitle,
} from "@/components/ui/dialog";
import { VisuallyHidden } from "radix-ui";

interface PlayerModalProps {
  tmdbId: string;
}

export function PlayerModal({ tmdbId }: PlayerModalProps) {
  const [open, setOpen] = useState(false);

  useEffect(() => {
    const handler = () => setOpen(true);
    document.addEventListener("kawakumo:open-player", handler);
    return () => document.removeEventListener("kawakumo:open-player", handler);
  }, []);

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogPortal>
        <DialogOverlay />
        <DialogContent className="max-w-4xl p-0 overflow-hidden">
          <VisuallyHidden.Root>
            <DialogTitle>Movie Player</DialogTitle>
          </VisuallyHidden.Root>
          <iframe
            src={`https://www.vidking.net/embed/movie/${tmdbId}`}
            width="100%"
            height="600"
            frameBorder="0"
            allowFullScreen
            title="Movie Player"
          />
        </DialogContent>
      </DialogPortal>
    </Dialog>
  );
}
```

**Note**: The `DialogPortal` will need its `container` prop pointed at the shadow root container, since the default portal target (`document.body`) is outside the shadow DOM. We'll pass the shadow root container ref through to the component.

### Step 7: Create content script CSS

Create `entrypoints/content.css` with two sections:
1. Host page styles (for the native play button, minimal, no Tailwind)
2. Tailwind import for the shadow root (used by shadcn components)

```css
/* Host page button styles (injected into host, not shadow DOM) */
.kawakumo-play-item {
  display: inline-flex;
  align-items: center;
}

.kawakumo-play-button {
  background-color: #1a1a2e;
  color: #ffffff;
  border: none;
  padding: 8px 16px;
  border-radius: 6px;
  font-size: 14px;
  font-weight: 600;
  cursor: pointer;
  transition: background-color 0.2s;
}

.kawakumo-play-button:hover {
  background-color: #16213e;
}
```

For the shadow root styles, WXT's `createShadowRootUi` accepts a `css` option where we can inline the Tailwind/shadcn styles. Alternatively, we create a separate CSS file that gets imported into the shadow root. The approach:

```tsx
// In the createShadowRootUi call, reference the CSS for the shadow root:
const ui = await createShadowRootUi(ctx, {
  name: "kawakumo-player",
  position: "inline",
  css: shadowStyles, // imported as ?inline CSS
  // ...
});
```

We'll import the shadcn styles as a string:
```tsx
import shadowStyles from "@/styles/shadow.css?inline";
```

Create `styles/shadow.css`:
```css
@import "tailwindcss";

@custom-variant dark (&:is(.dark *));

@theme {
  /* same shadcn theme variables as popup/style.css */
}
```

### Step 8: Full content script assembly

```tsx
// entrypoints/content.tsx
import ReactDOM from "react-dom/client";
import { PlayerModal } from "@/components/PlayerModal";
import shadowStyles from "@/styles/shadow.css?inline";
import "./content.css";

export default defineContentScript({
  matches: ["*://*.themoviedb.org/movie/*"],
  async main(ctx) {
    const tmdbId = getTmdbId();
    if (!tmdbId) return;

    injectPlayButton(tmdbId);
    await mountPlayerModal(ctx, tmdbId);
  },
});

function getTmdbId(): string | null {
  const lastSegment = window.location.pathname.split("/").pop();
  if (!lastSegment) return null;
  const id = lastSegment.split("-")[0];
  return /^\d+$/.test(id) ? id : null;
}

function injectPlayButton(tmdbId: string) {
  const targetUl = document.querySelector(
    "#original_header > div.header_poster_wrapper.true > section > div.flex.flex-col > ul"
  );
  if (!targetUl) return;

  const li = document.createElement("li");
  li.className = "kawakumo-play-item";

  const button = document.createElement("button");
  button.textContent = "▶ Play Movie";
  button.className = "kawakumo-play-button";
  button.addEventListener("click", () => {
    document.dispatchEvent(new CustomEvent("kawakumo:open-player"));
  });

  li.appendChild(button);
  targetUl.appendChild(li);
}

async function mountPlayerModal(
  ctx: ContentScriptContext,
  tmdbId: string,
) {
  const ui = await createShadowRootUi(ctx, {
    name: "kawakumo-player",
    position: "inline",
    css: shadowStyles,
    onMount(container) {
      const root = ReactDOM.createRoot(container);
      root.render(<PlayerModal tmdbId={tmdbId} container={container} />);
      return root;
    },
    onRemove(root) {
      root?.unmount();
    },
  });
  ui.mount();
}
```

## Notes

- **Hybrid DOM approach**: The play button is a native DOM element in the host page (so it fits naturally in TMDB's `<ul>`), while the modal lives in an isolated shadow root (so shadcn/Tailwind styles work without leaking)
- **Custom event bridge**: The native button communicates with the shadow root React app via `CustomEvent("kawakumo:open-player")` dispatched on `document`
- **Shadow root CSS**: Imported via `?inline` suffix so it's passed as a string to `createShadowRootUi`'s `css` option, which injects it inside the shadow DOM
- **DialogPortal container**: Must point to the shadow root container element, not `document.body`, otherwise the dialog renders outside the shadow DOM and loses its styles
- All host page CSS classes are prefixed with `kawakumo-` to prevent collisions with TMDB styles
